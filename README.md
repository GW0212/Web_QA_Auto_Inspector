# Web QA Auto Inspector

URL 기반 웹사이트 자동 QA 리포트 생성 대시보드입니다.

## 프로젝트 개요

공개 웹페이지 주소를 입력하면 페이지가 정상적으로 열리는지, 이미지와 링크가 깨지지 않았는지, 모바일 화면이 보기 좋은지, 페이지 오류와 속도·보안·입력칸 안내 문제가 없는지 자동 점검하는 포트폴리오용 QA 자동화 웹 애플리케이션입니다.

현재 버전은 서버 없이 동작하는 프로토타입 검사 엔진으로 구성되어 있으며, 추후 Playwright 기반 실제 검사 로직을 연결하기 쉽도록 결과 데이터 구조를 분리했습니다.

## 주요 기능

- URL 입력 및 형식 검증
- 예시 URL 버튼 제공
- QA 검사 진행 상태 표시
- PASS / WARN / FAIL 결과 표시
- Critical / Major / Minor 심각도 분류
- 전체 QA 점수 자동 산정
- 우선 수정 권장 이슈 자동 정리
- 항목별 상세 리포트 접기/펼치기
- JSON 결과 다운로드
- HTML 리포트 다운로드
- QA 요약 복사
- 인쇄 창을 통한 PDF 저장
- 체감 속도 데모 점검
- 보안 연결 점검
- 기본 보안 설정 점검
- 키보드 조작 / 입력칸 안내 점검
- 페이지 구조 / 모바일 표시 기준 점검
- 새 창 외부 링크 안전성 점검
- 브라우저 사전 연결 확인 기반 접속 판정 보완
- 섹션 메뉴 클릭 시 상단 헤더 아래로 정확히 정렬되는 스크롤 이동
- 라이트/다크 모드 지원
- 모바일 반응형 UI 지원

## 실행 방법

별도 설치 없이 `index.html` 파일을 브라우저에서 열면 됩니다.

로컬 서버로 실행하고 싶다면 아래 명령어를 사용할 수 있습니다.

```bash
npx serve .
```

또는 VS Code의 Live Server 확장 프로그램으로 실행해도 됩니다.

## 파일 구조

```text
index.html
styles.css
app.js
package.json
README.md
assets/
  favicon.svg
```

## 추후 Playwright 연동 방향

현재 `app.js`의 `createDemoReport(url)` 함수가 프로토타입 QA 결과를 생성합니다. 실제 자동화 검사로 확장할 때는 이 함수를 Node.js + Express API로 분리하고, 서버에서 Playwright를 실행해 동일한 형태의 JSON을 반환하도록 구성하면 됩니다.

예시 확장 구조:

```text
server/
  index.js
  playwrightRunner.js
  reportMapper.js
```

검사 흐름:

1. 프론트엔드에서 URL 입력
2. `/api/inspect`로 URL 전달
3. 서버에서 Playwright 실행
4. 페이지 접속, 콘솔 에러, 이미지/링크, SEO/접근성 항목 수집
5. 현재 리포트 JSON 스키마로 변환
6. 프론트엔드 대시보드에 표시

## v17 점검 반영 사항

- Dashboard / Test Items / Report 메뉴 클릭 시 각 섹션 제목이 상단 헤더 아래에 깔끔하게 보이도록 스크롤 위치를 보정했습니다.
- 모바일 헤더, 메뉴, 카드, 요약 영역의 반응형 레이아웃을 보완했습니다.
- 브라우저 no-cors 사전 연결 확인을 추가해 접속 가능 여부 판정 신뢰도를 높였습니다.
- GitHub Pages 계열 공개 페이지에 대한 데모 판정 프로파일을 보정해 무작위 FAIL이 과도하게 발생하지 않도록 개선했습니다.
- 문서 구조 / 뷰포트 검사와 외부 링크 보안 검사를 추가했습니다.
- 점수 산정 방식을 가중치 기반으로 조정해 Critical/Major 이슈는 강하게, Minor 개선사항은 과도하게 감점되지 않도록 수정했습니다.
- 압축 해제 시 프로젝트 파일이 바로 보이도록 루트 구조를 유지했습니다.

## v17 추가 개선

- 상세 리포트의 검사 내용, 판정 기준, 실제 결과, 개선 방법 문구를 비개발자도 이해하기 쉬운 표현으로 정리했습니다.
- `alt`, `href`, `Core Web Vitals`, `보안 헤더`, `크롤링`처럼 어려운 용어는 화면에서 쉬운 설명 중심으로 풀었습니다.
- Test Items 카드 설명도 전문 용어 중심에서 실제 사용자 관점의 설명으로 수정했습니다.

## v18 추가 개선

- 모바일에서 HTML 리포트를 PDF로 저장했을 때 항목별 결과가 세로로 쪼개져 보이는 문제를 해결했습니다.
- PDF/HTML 리포트의 항목별 결과 표를 카드형 리포트로 변경해 작은 화면에서도 핵심 결과와 개선 방법을 정상적으로 읽을 수 있도록 수정했습니다.
- 인쇄/PDF 저장 전용 스타일을 추가해 A4 기준으로 점수, 요약 수치, 우선 수정 이슈, 항목별 결과가 안정적으로 배치되도록 개선했습니다.

## v19 수정 사항

- 모바일에서 `PDF 저장` 클릭 시 빈 화면이 뜨던 문제를 해결했습니다.
- PDF 저장은 새 창/팝업 방식이 아니라 현재 페이지 안에 인쇄 전용 리포트를 생성한 뒤 `window.print()`를 실행하도록 변경했습니다.
- 모바일 브라우저에서 `HTML 리포트` 버튼이 반응하지 않는 문제를 방지하기 위해 HTML 리포트를 새 탭으로 열도록 수정했습니다.
- HTML 리포트 새 탭에는 `PDF 저장/인쇄` 버튼을 추가했습니다.
- PDF/HTML 리포트 모두 모바일에서도 표가 세로로 쪼개지지 않도록 카드형 리포트 구조를 유지했습니다.


## v20 수정 사항

- PC에서 HTML 리포트 버튼이 동작하지 않던 문제를 수정했습니다.
- PC에서 PDF 저장 버튼이 동작하지 않던 문제를 수정했습니다.
- 리포트 생성 함수에서 잘못된 요약 데이터 참조를 수정했습니다.
- HTML/PDF 리포트 생성 중 오류가 발생해도 조용히 멈추지 않고 안내 메시지가 표시되도록 보완했습니다.
- zip 압축 해제 시 바로 프로젝트 파일이 보이도록 루트 구조를 유지했습니다.
